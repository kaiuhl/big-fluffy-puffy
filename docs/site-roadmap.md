# Site Roadmap

## Purpose

The website should make fireless camping feel prepared, social, and normal. Early pages can stay sparse, but each one should make the organization easier to understand, trust, or join.

## Phase 1: Credible Placeholder

Live now:

- Nonprofit name
- Mission line
- Short explanation
- More-soon status
- Austere black/white visual system

Near-term refinements:

- Add a contact route or email link once the public inbox exists.
- Add one strong photo or looping video when there is original media worth using.
- Add basic Open Graph metadata for link previews.
- Add analytics only if they are privacy-respecting and useful.

## Phase 2: Public Identity

Add the first real content pages:

- `About`: what BFP is, why it exists, where it starts.
- `Why Fireless`: practical fire risk, restrictions, smoke, warmth, and culture.
- `Field Notes`: short posts about fireless trips, gear habits, restrictions, and local conditions.
- `Contact`: one clear way to reach the organization.

Keep the voice practical. Avoid sounding like a grant application or outdoor brand campaign.

## Phase 3: Useful Tools

Start turning the site into a service:

- Regional fire restriction status pages.
- Source pages for national forests, parks, counties, and agencies.
- Change history for restrictions over time.
- Email/RSS alerts for selected areas.
- A small "camp without fire" checklist.

The first version can be read-only and manually curated. Automation can arrive source by source.

## Phase 4: Programs

Once the nonprofit has real-world work, add program pages:

- Fireless camp kits
- Trailhead/permit-office education materials
- Partner campground signage
- Group camping pledge or certification
- Volunteer/source monitor program

Program pages should show concrete actions and outcomes, not vague intention.

## Content Model

Likely early content types:

- Pages: durable explainers and organization info
- Field notes: dated narrative or practical posts
- Sources: agencies or places monitored for fire restrictions
- Restriction snapshots: parsed state at a point in time
- Alerts: subscriptions or outbound notices

Bridgetown can own pages and field notes. Roda/Postgres should own sources, snapshots, and alerts.

## Homepage Evolution

The homepage can grow in bands:

- Nameplate and mission
- One large visual moment
- "Why fireless" statement
- Current regional fire-risk/restriction signal
- Three practical habits
- Latest field note
- Contact/join prompt

Resist filling the homepage with every future idea. Let it act like a clear front door.

## Visual Assets To Gather

Prioritize original or specific imagery:

- Puffy jackets and quilts in real camp conditions
- Headlamps, lanterns, mugs, cold hands, group seating
- Damp Pacific Northwest woods
- Smoky or high-fire-risk landscape only when editorially necessary
- Agency signage and restriction notices, photographed clearly

Avoid generic campfire imagery as the default hero. The point is to make the absence of fire feel intentional.
